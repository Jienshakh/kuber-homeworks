# Расчёт ресурсов для Kubernetes кластера

## 1. Исходные данные

| Компонент | Количество реплик | RAM на реплику | CPU на реплику | Итого RAM | Итого CPU |
|-----------|-------------------|----------------|----------------|-----------|-----------|
| База данных | 3 | 4 ГБ | 1 | 12 ГБ | 3 |
| Кеш | 3 | 4 ГБ | 1 | 12 ГБ | 3 |
| Бекенд | 10 | 0.6 ГБ | 1 | 6 ГБ | 10 |
| Фронтенд | 5 | 0.05 ГБ | 0.2 | 0.25 ГБ | 1 |
| **Всего** | **21** | | | **30.25 ГБ** | **17** |

## 2. Выбор количества нод

Для отказоустойчивости базы данных и кеша (по 3 реплики) необходимо минимум 3 ноды, чтобы при выходе одной ноды из строя оставшиеся две реплики могли обеспечить кворум.

## 3. Распределение подов по нодам

При равномерном распределении на одной ноде окажется:
- 1 под БД (4 ГБ / 1 CPU)
- 1 под кеша (4 ГБ / 1 CPU)
- 3-4 пода бекенда (1.8-2.4 ГБ / 3-4 CPU)
- 1-2 пода фронтенда (0.05-0.1 ГБ / 0.2-0.4 CPU)

Максимальная загрузка одной ноды (без учёта служебных ресурсов):
- RAM: 4 + 4 + 2.4 + 0.1 = 10.5 ГБ
- CPU: 1 + 1 + 4 + 0.4 = 6.4 ядра

## 4. Расчёт служебных ресурсов

Для ноды с 18 ГБ RAM (формула из статьи):
- 25% от первых 4 ГБ = 1 ГБ
- 20% от следующих 4 ГБ = 0.8 ГБ
- 10% от следующих 8 ГБ = 0.8 ГБ
- 6% от оставшихся 2 ГБ = 0.12 ГБ
- Итого для kubelet и OS: ~2.7 ГБ
- + порог вытеснения (eviction threshold): ~0.1 ГБ
- Всего служебных: ~2.8 ГБ

Для CPU (18 ГБ нода → ~10 ядер):
- 6% от первого ядра = 60m
- 1% от второго = 10m
- 0.5% от следующих двух = 10m
- 0.25% от следующих шести = 15m
- Итого служебных: ~95m

## 5. Запас на отказ одной ноды

При выходе одной ноды из строя все поды перераспределяются на две оставшиеся:

- Нагрузка на одну оставшуюся ноду: 30.25 / 2 = 15.125 ГБ (поды)
- С учётом служебных: 15.125 + 2.8 = 17.925 ГБ
- По CPU: 17 / 2 = 8.5 ядра (поды) + служебные

## 6. Итоговые параметры нод

Учитывая, что облачные провайдеры предлагают дискретные размеры инстансов, выбираем ближайшие доступные значения с запасом:

**Количество нод: 3**

**Параметры одной ноды:**
- RAM: 18 ГБ
- CPU: 10 ядер

**Проверка:**
- При нормальной работе загрузка ноды: ~13.3 ГБ (10.5 поды + 2.8 служебные) — запас 26%
- При отказе одной ноды загрузка оставшихся: ~17.9 ГБ — запас 0.5% (критично, но допустимо для кратковременной нагрузки до восстановления ноды)
- По CPU запас достаточный даже при отказе ноды

## 7. Допущения

Расчёт произведён на основе следующих допущений:
- Учитываются только ресурсы, указанные в задании (БД, кеш, бекенд, фронтенд)
- Ресурсы, необходимые для работы DaemonSet, не учитывались, так как о них нет информации в задании
- Расчёт служебных ресурсов (kubelet, OS) выполнен по формуле из статьи https://learnkube.com/kubernetes-node-size
- Подразумевается, что все ноды однотипные (универсальные), так как требования к разным типам нод в задании не описаны
